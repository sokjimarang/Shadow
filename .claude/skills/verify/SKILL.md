# Self-Verification Skill

작업 완료 후 스스로 검토를 수행합니다.

---

## 검토 항목

다음 항목들을 하나씩 점검하세요:

### 1. 완료 여부 확인
- 사용자가 요청한 모든 작업이 실제로 완료되었는가?
- 빠뜨린 요구사항은 없는가?
- "나중에 하겠다"고 미룬 것은 없는가?

### 2. 임시 해결책 점검
- 근본 원인을 해결했는가, 아니면 증상만 숨겼는가?
- workaround나 hack으로 처리한 부분은 없는가?
- TODO, FIXME, 임시 코드가 남아있지 않은가?

### 3. 의도 반영 확인
- 사용자의 원래 의도를 제대로 이해하고 구현했는가?
- 요청과 다르게 해석한 부분은 없는가?
- 사용자가 명시하지 않았지만 당연히 기대할 만한 것을 놓치지 않았는가?

### 4. 테스트 검증
- 관련 테스트가 실행되었는가?
- 새로 작성한 코드에 대한 테스트가 필요하지 않은가?
- 테스트가 실제로 통과했는가? (실패를 무시하지 않았는가?)

### 5. 부작용 확인
- 기존 기능을 망가뜨리지 않았는가?
- 예상치 못한 side effect는 없는가?

---

## 조치

위 항목 중 하나라도 문제가 있다면:
1. 문제점을 명확히 밝히고
2. 즉시 수정 작업을 진행하세요

모든 항목이 통과되면 검토 결과를 요약해서 보고하세요.
